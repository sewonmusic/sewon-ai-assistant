## 1. 프로젝트 개요 (Project Overview)
본 프로젝트는 온·오프라인 악기 판매점 **세원뮤직**의 비즈니스 효율성을 극대화하기 위한 **통합 업무 자동화 및 AI 비서(Gem) 구축 프로젝트**입니다. 
파이썬 스크립트 기반의 데이터 파이프라인과 Anthropic Claude API의 추론/비전 능력을 결합하여, 이카운트(Ecount) ERP 중심의 재고 동기화, 다채널 쇼핑몰 관리, 그리고 향후 사내 지식 기반 RAG 챗봇까지 아우르는 확장성 있는 아키텍처를 설계합니다. 

모든 데이터베이스와 최종 결과물은 로컬 보안을 원칙으로 하며, 사용자의 '바이브 코딩' 프로세스에 맞춰 모듈 단위로 개발을 진행합니다.

---

## 2. 시스템 아키텍처 및 인프라 (Infrastructure)
* **하드웨어:** M4 Mac Mini (32GB Unified Memory)
* **소프트웨어 스택:** Python (venv 가상환경), Anthropic Claude API (주요 추론 및 Vision 엔진), Obsidian (메인 UI, 에디터 및 체크리스트 관리)
* **데이터베이스:** 로컬 SQLite (`database/sewon_mapping.db` 파일 기반)
* **보안 및 제약사항:** 
  * 외부 API(Claude) 전송 시 민감한 B2B 데이터 보안 통제 및 프롬프트 최적화
  * API 토큰 비용 절감을 위한 캐싱 및 방어 로직 구현
  * 모든 DB의 Read/Write 및 최종 파일 저장은 Mac 로컬 환경에서 수행

## 3. 핵심 비즈니스 규칙 (Core Business Rules)
1. **재고 데이터 무결성 (Single Source of Truth):**
   * 모든 상품 매핑과 재고 계산의 절대 기준은 **'이카운트 단일 품목(SKU)'**으로 고정합니다.
   * 이카운트에 존재하지 않는 품목은 시스템 테이블에 등록될 수 없습니다.
2. **휴먼 인 더 루프 (Human-in-the-loop):**
   * AI의 자동 매핑 추론 결과는 DB에 즉시 반영되지 않고 `status = 'PENDING'`으로 저장됩니다.
   * 승인 대기 건은 옵시디언 내 `Mapping_Review_Pending.md` 파일로 자동 생성됩니다.
   * 관리자가 눈으로 확인 후 승인(체크)한 항목만 SQLite DB에 `status = 'APPROVED'`로 최종 반영됩니다.
3. **확장성을 고려한 OCR 처리 대기:**
   * 향후 이미지 형태의 거래처 단가표 입수에 대비하여, Claude Vision API를 활용한 텍스트 구조화 파이프라인 연결을 시스템 아키텍처에 미리 고려해 둡니다.

## 4. 디렉토리 구조 및 데이터 파이프라인 (Refactored)
프로젝트는 기능별 모듈화 및 데이터 흐름에 따라 다음과 같이 구성됩니다.

* `src/`: 핵심 파이썬 소스 코드 (모듈별 분리)
  * `kakao_collector/`: 카카오톡 대화록 파싱 및 미디어 매칭 파이프라인 (Phase 1)
* `database/`: 로컬 SQLite DB 파일 보관 (Phase 1.5 - `sewon_mapping.db` 등)
* `local_data/`: 파이프라인을 거치는 임시/로컬 데이터
  * `00_raw/`: 수동 백업된 카카오톡 대화록(.csv) 및 미디어 원본 보관
  * `01_preprocessed/`: 날짜별 그룹화 파싱 및 미디어 파일이 정렬되는 임시 저장 폴더
  * `03_archive/`: 처리가 완료된 문서/파일 등의 백업 아카이빙 (원본 대화록 CSV는 용량 관리를 위해 즉시 삭제)
* `obsidian_vault/`: 최종 결과물 저장 및 사용자 UI
  * `02_journals/`: LLM이 요약한 마크다운 업무 일지(.md) 보관
  * 기타 검토용 마크다운 파일(예: `Mapping_Review_Pending.md`) 생성 공간

## 5. 데이터베이스 스키마 설계 (Phase 1.5 기준)
무결성 유지와 유연한 가격/채널 관리를 위해 `database/sewon_mapping.db`는 2개의 핵심 테이블로 정규화하여 설계합니다.

### 5.1. `product_master` (기준 정보 테이블)
이카운트 ERP 기반의 마스터 데이터입니다.
* `id`: PK (Primary Key)
* `ecount_sku`: 이카운트 품목코드 (절대기준값, Unique)
* `product_name`: 상품명
* `purchase_price`: 매입가(원가)
* `sale_price`: 기본 판매가
* `updated_at`: 최종 업데이트 일시

### 5.2. `channel_mapping` (채널 연결 테이블)
각 쇼핑몰(스마트스토어, 쿠팡, 샵링커 등)과 마스터 데이터를 연결합니다. 정산 매칭을 위해 채널별 코드를 반드시 수집합니다.
* `id`: PK
* `ecount_sku`: FK (`product_master` 참조)
* `channel_name`: 쇼핑몰 또는 연동 솔루션명 (예: 스마트스토어, 쿠팡, 샵링커, 지마켓 등)
* `channel_product_code`: 쇼핑몰 자체 부모 상품코드
* `channel_option_code`: 쇼핑몰 자체 자식/옵션 코드 (필수값)
* `status`: 승인 상태 (`PENDING`, `APPROVED`, `REJECTED`)

## 6. 단계별 로드맵 (Phases)

### ✅ [완료] Phase 1: 비정형 데이터 자산화 파이프라인 (`src/kakao_collector`)
* **목표:** 카카오톡 대화록의 날짜별 그룹화 파싱 및 미디어 파일의 일괄 매칭
* **핵심 기능:** Claude API를 활용한 텍스트/비전 데이터 요약 (`kakao_collector` 하위의 파이썬 모듈 활용)
* **결과물:** YAML Front Matter 및 명사형 종결어미 요약이 포함된 구조화된 마크다운(`.md`) 문서를 옵시디언 볼트(`obsidian_vault/02_journals`)에 자동 저장
* **참고 (검토된 데이터 샘플):**
  * 거래처: 데임악기 (김영준, 조성찬)
  * 대화 특징: 위탁 배송(직발송) 요청 정보, 송장 번호 공유, 신제품 입고 리스트 파일(`SIRE V NewGen(가격포함).xlsx`) 및 재고 문의 파일(`Sire_Guitar_Models.xlsx`) 교환이 빈번함.

### 🔄 [현재 진행] Phase 1.5: Ecount 통합 매핑 DB 및 채널 정규화
* **목표:** 이카운트 ERP 무료 API 연동 및 SQLite 기반 통합 매핑 DB 구축 (`database/sewon_mapping.db`)
* **핵심 과제:** 다채널 쇼핑몰(스마트스토어, 쿠팡 등)과 거래처 엑셀 간의 상품명/상품코드(SKU) 불일치 문제 해결 및 데이터 정규화
* **진행 방식:** 모듈별 코딩을 통한 파이프라인 구축 및 옵시디언 기반의 휴먼 인 더 루프 검증 시스템 활성화

### ⏳ [예정] Phase 2: RAG 비즈니스 챗봇 도입
* **목표:** ChromaDB 기반 벡터 검색 및 Streamlit 웹 UI 구축
* **핵심 과제:** 옵시디언에 축적된 Phase 1의 업무일지 및 상품 위키 문서를 인덱싱하여, 즉각적인 질의응답이 가능한 사내 지식 저장소 완성

### ⏳ [예정] Phase 3: 마케팅 및 재무 분석 자동화
* **목표:** 유료 광고 채널 API 연동 및 마진율 분석
* **핵심 과제:** 네이버, 쿠팡 등 광고 API 연동, 상품별 실제 ROI 분석 및 대시보드 시각화
