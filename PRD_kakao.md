# 📋 [Sub-PRD 1] 카카오톡 대화 및 파일 전처리 자동화 (수동 수집 기반)

## 1. 프로젝트 개요
* **상위 프로젝트:** 세원뮤직 전용 AI 업무 관리 비서
* **하위 프로젝트 목표:** 카카오톡 공식 API 부재 및 GUI 자동화의 잦은 오류로 인해, 사용자가 수동으로 다운로드한 카카오톡 대화록과 파일들을 AI 비서가 알아서 분류하고 정제하는 "전처리 자동화" 파이프라인 구축.
* **대상 범위:** 카카오톡에서 수동으로 추출한 대화록(.csv 또는 .txt), 공유 파일, 이미지 및 동영상
* **최종 저장소:** 임시 폴더(`[00_수동다운로드_RAW]`)를 거쳐, 로컬 환경의 `[01_정리전_대화및파일]` 폴더 내 거래처별 디렉토리로 정렬

---

## 2. 상세 워크플로우 (수동 수집 및 전처리 로직)

### 단계 A: 데이터 수동 백업 (User Action)
1. 사용자가 Mac 카카오톡에서 거래처 채팅방의 **대화 내용 내보내기**(.csv 또는 .txt)를 수행합니다.
2. 채팅방 서랍의 **[사진/동영상]**, **[파일]** 탭에서 필요한 미디어 및 문서 파일을 로컬 다운로드 폴더(`[00_수동다운로드_RAW]`)로 다운로드합니다.

### 단계 B: 데이터 스캔 및 분류 (Auto Preprocessing)
1. Python 스크립트가 `[00_수동다운로드_RAW]` 폴더를 주기적으로 모니터링하거나 사용자가 스크립트를 수동으로 실행합니다.
2. 스크립트가 대화록의 파일명과 내용 상단, 혹은 다운로드된 파일들을 분석하여 **거래처명과 날짜**를 식별합니다.
3. 식별된 정보를 바탕으로 `[01_정리전_대화및파일]/[YYYY-MM-DD_거래처명]` 폴더를 자동 생성하고, 매칭되는 대화록과 파일들을 해당 폴더로 이동시킵니다.

### 단계 C: 대화록 정제 및 링크 추출 (Auto Processing)
1. **[대화록 필터링]**: 대화록 원본 파일에서 필요한 대화(예: 특정 날짜의 대화)만 필터링하여 불필요한 시스템 메시지 및 이전 날짜의 기록을 제거합니다.
2. **[링크 추출 및 메타데이터 수집]**: 필터링된 대화록에서 `http` / `https` URL 패턴을 식별합니다. Python 크롤링 라이브러리(BeautifulSoup 등)를 사용하여 해당 URL의 OpenGraph 태그(제목, 설명) 정보를 가져와 `링크목록_YYYYMMDD.txt` 등으로 요약 저장합니다.

---

## 3. 파일 저장 및 아카이빙 규칙
* 전처리 스크립트가 작동을 완료하면 데이터는 거래처별/날짜별로 격리되어 아래 구조로 안전하게 보관됩니다.

```text
[01_정리전_대화및파일] / 
    └── [YYYY-MM-DD_거래처명] /
            ├── 대화록_YYYYMMDD.txt (또는 .csv, 필요 내용만 필터링된 파일)
            ├── 링크목록_YYYYMMDD.txt (공유된 웹사이트 제목 및 URL 주소 메타데이터)
            ├── [사진_동영상] /
            │     └── (수동 수집 후 스크립트가 자동 이동시킨 스크린샷, 제품 이미지 등)
            └── [문서_파일] /
                  └── (수동 수집 후 스크립트가 자동 이동시킨 엑셀 재고표, 카탈로그 등)
```
