1. 테스트 인증키 발급
• 인증키 발급 전 테스트 인증키를 이용하여 요청 내용에 대한 검증을 받아야 합니다.
• ERP에 로그인 (https:/login.ecount.com)하여 SelfiCustomizing> 정보관리 API인증키발급 > API인증현황 > 키발급 버튼에서 인증키를 발급합니다.
• 발급한 인증키는 2주간 유효합니다.
※ 검증을 통한 실제 인증이라 요청을 하지 않고 테스트 인증키를 이용하여 지속적으로 요청이 있는 경우 통보 없이 차단될 수 있습니다.
• 인증키 발급은 최대 3회까지 갱신이 가능하며 추가 갱신은 요청 후 검토하여 진행이 가능합니다.
2. IP등록
• 발급한 인증키로 검증하기 위해서는 OAPI를 사용할 IP를 등록해야 합니다.
• ERP에 로그인하여 SelfCustomizing > 정보관리 > API인증키발급 > API인증현황 > [IP등록] 버튼에서 IP를 등록합니다.
• IP는 최대 5개 까지 등록 가능합니다.
3. 개발 검증
0A의 검증은 보는 항볼이 아닌 사용하려는 식기반 검증을 말으시면 됩니다
• 이카운트에서 테스트 인증키로 요청한 내용을 분석하여 적합한 경우 자동으로 검증을 진행합니다.
※ 검증은 요청과 동시에 진행됩니다.
• 테스트 URL http://sboapi.ecount.com에 요청을 진행하여 전송 성공 시 검증은 완료됩니다.
※ API 직접실행도 URL http:/sboapi.ecount.com 페이지에서 진행 시 테스트 인증키로 로그인이 가능합니다.
• 검증은 요청한 API에 한정하여 진행되며 추가 AP를 사용하길 원하시면 테스트 인증키를 사용하여 추가 요청을 진행 바랍니다.
• 검증된 AP리스트는 SeliCustomizing> 정보관리 > API인증키발급 > AP인증현황에서 확인할 수 있습니다.
4. 인증키 발급
• 이카운트 API와 연동을 위해 인증키를 발급받아야 합니다.
• ERP에 로그인 (https://ogin.ecount.com) 하여 SelfCustomizing> 정보관리 API인증키 발급화면에서 인증키를 발급합니다.
• 발급한 인증키는 1년간 유효합니다.
5. Open API 사용
• 이카운트 Open API를 호출하기 위해서는 세션 ID가 필요하며, 세션 ID는 로그인 API를 통해서 발급할 수 있습니다.
• 발급받은 세션 ID는 설정한 시간 (ERP로그인> SeliCustomizing> 환경설정> 보안설정 자동로그아웃 설정) 동안 사용 가능하며, 주문 API 또는 판매 API를 한 번 호출하면 다시 설정 시간 동안 사용할 수 있습니다.
※ 설정한 시간 동안 주문 API, 판매 API 등을 호출하지 않으면 해당 세션 ID는 더 이상 유효하지 않으며 로그인 API를 통해 다시 발급받아야 합니다.
• 주문, 판매 등 입력API를 이용하기 위해서는 ERP웹자료올리기에 사용하려는 항목이 추가되어있어야 합니다.
※ 웹자료올리기 설정은 각 입력메뉴 > 하단 웹자료올리기 버튼 > 자료올리기 항목추가로 가능합니다.
